# eval-kit — product

Product owns **what & why**: audience, problem, promise, boundaries, and when to use / when not to.
It is the contract the design layer reconciles to and is readable without the code.

Phase 0 only establishes the repository home for shared eval mechanics. Product detail should define
which evaluation jobs belong here, which remain consumer-specific, and what compatibility promises
the shared package makes.
